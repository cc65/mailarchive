Here's another bug:

When optimization is enabled and favour size is disabled (-Oi compiler
switch), using a constant long value that is less than 0x10000 creates invalid
assembler code in some places. The assembler will output an error when trying
to translate the generated code.

All these bugs do probably mean that not enough people are using cc65:-)
