I'm thinking about converting the cc65 sources to C++ for easier maintenance
and extendibility. Any objections against this idea?

People using binary versions shouldn't notice any difference. How many people
are actually compiling cc65 or even hacking the source?

The only reason against a C++ conversion I can think of is that cc65 will
never be able again to translate itself. This is no problem as long as the
only target is the 6502 since the memory footprint of cc65 is too big for a
6502 system anyway. This may change if the 816 is supported any time in the
far, far future.
