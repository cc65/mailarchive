Hi!

> I see that the cc65 compiler has support for:
>
> o  C= 64
> o  C= 128
>
> but how about support for the lib6502-object-file? As I understand it, the lib6502-object-file is a binary format that can be used across OS'es. Currently these OS'es support lib6502-object-file:
>
> o  LNG (LUnix Next Generation)
> o  CS/A65 (I can't remember the exact name)

I have started to add the o65 format (6502, executable only) to the linker
some time ago. I'm not sure if the o65 format itself is useful for many
people, but the idea was to lay the groundwork for more executable file
formats.

Support for the o65 format is 95% finished, but I have not worked on it for
months. However, to be really useful, a special C library would be needed that
makes use of the shared libraries.

I have currently not much time to work on the compiler package, so there will
be no news from me until at least next year. However, there may be other news,
and maybe a new version some time not too far in the future.
