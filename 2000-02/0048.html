> It also uses the wrong fillval for alignment (0x00 rather than
> whatever was specified in the linker Config file).

This works in my tests. Can you send me the linker config file that is
causing you problems?

> After it's moved, is it still necessary to also have an align=###
> directive in the linker Config file?

Yes, that is necessary. The alignment of the segment in the config file must
be at least as large as the alignments of the single segment sections in the
object files. 

It would be possible to drop that requirement for binary files, because the
linker knows the start address of the segment. But first, it would be more
confusing, and second, it will no longer work with relocatable formats like
o65, where the exact starting address is not known.
