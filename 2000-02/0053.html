I had cases of FTP server abuse in the last two days. Several clients
established dozens or even hundreds of simultaneous connections to download
complete directory contents, aborting after 100K and starting over. This
caused a lot of trouble for me and forced me to add filters for these clients.

Since ftp.musoftware.de has only a small internet connection (which is
entirely paid by me), I'm sorry but I cannot tolerate these things. So I had
to setup tighter limits for FTP and other connections that may cause trouble
for some of you in the future:

  * The ftp server does not accept more than two concurrent logins.

  * A RFC 822 compliant password is enforced. This means that you can no
    longer use login passwords like "IE30User@". Please fix you browser setup
    if you have problems.

  * Idle timeout is 90 seconds. Several WWW browsers including IE and Netscape
    are broken and don't close an FTP connection, tying up server resources.

  * Connections from hosts without a valid host name reverse mapping are not
    accepted.

  * Domains that cause trouble are blacklisted.

I'm very sorry if this causes any inconveniences, but it helps to keep the
server alive.
