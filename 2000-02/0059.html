> There are arguments for filling the RUN area both with FILLVAL and
> with the RUN data.  I think I would fill it with FILLVAL.  POLA.
[...]
> I don't think it's necessary to write-out type=bss segments.  There's
> not much that makes a bss segment to be a bss segment, otherwise.  :)

After sleeping on it, I would say, the cleanest approach is to write the
RUN area as well as the BSS segments if "fill = yes" is specified. Not
writing these segments to the file will create holes (and therefore
problems), and you can easily avoid having the data written by not
specifying "fill = yes". So actually the RUN area (if it is given) is
handled as BSS style data.
