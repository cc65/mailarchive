Hello.

I'm proud to announce the first public release of GEOSLib.                

GEOSLib is currently separated from cc65 source tree but it is expected
soon to be integrated with cc65 as a new target system - GEOS. Work is
already in progress. Latest snapshots of cc65 have support for target
system 'geos'.

Most of GEOSLib is basically a wrapper for almost all GEOS Kernal
functions. It allows easy, fast and efficient cross-development for GEOS.
GEOSLib is something more than just a wrapper. It has custom functions for
standard windows (dialog boxes) with messages, file requester etc.
Supported target system is GEOS for both C64 and C128, but currently C128
version lacks few of its specific functions.

One of the biggest advantages of GEOSLib is that the output binary format
is .cvt, which allows easy transfer into GEOS environment. These files can
be easily converted into GEOS native format either by using Star
Commander, or native GEOS Convert v2.5.

GEOSLib is fully documented. It lacks examples but these will be
constantly added.

Currently GEOSLib can be easily used only under OSes which have 'bash' and
'awk' programs. For all others it will be a bit less comfortable.
This will change when resource compiler will be finished, then creating
menus, custom dialog boxes will be as easy as writting a simple resource
definition file.

At the present stage, GEOSLib doesn't inherit anything from cc65 standard
libraries. It will most probably be changed.

Send me any questions, comments, ideas, bugreports, etc. Thanks.
