On Mon, Jun 26, 2000 at 09:43:46AM -0500, David Lloyd wrote:
> That's OK... this looks like it's just what I need.  Can I count on the
> format of this output remaining the same in future versions?

I don't want to promise it, because it was just meant as debug output.
However, the probability that the object format changes is a lot higher, than
that this format changes. In fact, I have a few extensions in mind that
require a change of the object file format. So, while it's not a perfect
solution to use this output, it is better than accessing the internals of the
object files directly.

A perfect solution would be a utility to dump the internals of the object
files in a fixed format that is maintained together with the other parts of
the development system, but we have to live with what we have:-)
