On Mon, Jun 26, 2000 at 06:21:31PM +0200, Christian Groessler wrote:
> How about a library to access the object files? Then if the object
> format changes, simply relink with the new lib (or make it a shared
> lib) and the app understands the objects again.

This is difficult if the format of the object files is allowed to change. A
format change means you have to handle new data, and old data may have new
structure. To some degree, common code is already used: The format definitions
are contained in several shared modules that are part of the assembler, linker
and archiver. But I think it is not possible to write a library with a
consistent interface with the assumption that the underlying format may change
in almost any direction.

For existing file formats this may be different.
