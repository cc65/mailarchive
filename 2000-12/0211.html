I'm sorry, this is definitely an error in the assembler. 

Please, please submit things like this as a bug report and don't mention it in
passing somewhere in an email discussion. As a result this will not get fixed
for 2.6 because it's too late:-(

The new version is finished (provided there are no real sho stopper bugs) and
will probably out somewhere around the next weekend/start of next week. Adding
a fix for this problem would require more time than I currently have.

As a workaround for now, please use the global namespace operator:

bar	= 1
.proc	foo
.if	(::bar = 1)		; Global namespace override for ::bar
	.byte	0
.endif
.endproc
