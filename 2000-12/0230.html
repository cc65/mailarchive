> I mentioned
> in the other reply that I am able to read the files in now, but
> they do not work with a simple "run" commmand. Must I load
> them to an area of memory and then "sys" to the executable?

No, it should work with a simple "RUN". Standard programs for the CBMs have a
BASIC header that uses the SYS command to call the machine language part. You
should be able to see the BASIC header if you issue a LIST command after
loading the program.
