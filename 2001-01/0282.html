looks like you are trying to say "short-circuit evaluation", which is
correct.

> In this case, it elaborate the first part (the part before the &&) and then (only if this is

> expression is FALSE.

Exactly.

> In all things I prefer to use the "for" instead the "while" when I write a C program...

I prefer to use syntax which reflects the way I think of the solution,
rather than have pet preferred syntax. If I am incrementing a variable,
I use ++, but if I am adding 1 to it, as opposed to incrementing it, I
use += 1. A decent compiler generates the same code in either case.

I like my programs to indicate by their style what they are trying to
accomplish. The more hints I can give myself later, the better I like
it.
