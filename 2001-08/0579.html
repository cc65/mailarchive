haha ;=) well... i would ofcoz (well i do it already, but cant really
use it efficiently this way) use it to make stuff running as fast as
possible ;) a profiler can really help finding which function is the
real speed-killer and which is rather worth beeing optimized for
size... moreover with some effort it should be possible to make
profiling almost cycle-exact (+/- 2 cycles i think) and its a lot
easier to figure out the best/worst/average cyclecount of a function,
or even a block of functions, with a profiler ;) also it should be a
valuable tool for everyone supporting the code of the compiler itself
and the libraries.... plus it would be one more step to beeing able to
let cc65 do whatever you could expect a compiler to do ;=)

oh and btw... i have a pretty 'alpha' state stdio/dos lib for the c64

so i can clean this up and make a test-application etc... not to speak
