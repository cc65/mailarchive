The emulators have disassemblers in them and so don't really
require swapping between 'C' and 'asm' files. Typically if you're
building an app then the C file is compiled such that the S file
is deleted after being assembled (i.e. CL65 rather than CC65).
Therefore it wouldn't be available to the emulator :-(
