On Fri, Aug 03, 2001 at 04:11:10AM -0400, Keates, Mark wrote:
> Appologies, I hadn't noticed that info in a label file before
> as I wasn't using the -g option. What does the 'al' mean before
> the address in the label file? 

"al" is a VICE command.

> Could the assembler flag any
> address that is loaded or stored to such that the label file
> could either only list labels or use a different symbol
> for constants?

I wouldn't distinguish this by the use of a symbol, but by it's declaration.
Symbols defined with a colon

        MyLabel:

would be marked as labels, while symbols defined with "=" would be marked as
constants. In addition, it would probably be a good idea to merge all the info
(file and line info, labels, other symbols) into one file and remove emulator
specific things like the "al" for VICE. See my notes on cleanup before.
