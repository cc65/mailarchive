I'm writing a program that uses it's own character-set. The location of the
character set must be aligned to 1024 bytes blocks, so it cannot be
allocated with malloc().

compiler puts other static data in that memory that gets destroyed by my
characters.
Is there a way to tell the compiler to leave some memory unused for these
kind of things?
