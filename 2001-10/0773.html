Hi all!

I just wanted to announce that version 0.4.1 my lwIP TCP/IP stack [http://www.sics.se/~adam/lwip/] can be compiled under cc65 and run on a C64. Note that this is not the same TCP/IP stack (the uIP stack) that was announced to be working with cc65 a while ago.

The lwIP stack is rather large compared to the uIP stack; the compiled code is around 40k and it uses up all avaliable RAM. It consists of 19 C source code files. I have tested the code only in a C64 emulator and not on the real thing. 

I have also compiled the lwIP stack for the 16-bit Mitsubishi M16c CPU with the commercial IAR C compiler (M16C/6x Embedded Workbench Target Descriptor version 1.33A) and it is quite fun to note that the code is nearly twice as large when compiled with the IAR compiler!
