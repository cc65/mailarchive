Technically speaking it it now tomorrow morning (00:20 local time) and I've
placed the new packages on the FTP server (they are named beta2). This version
should fix the problems with illegal ranges for the Watcom compiled binaries
(and probably some others caused by the macros used to inline code).

I've also created new RedHat RPMs because of a few more changes:

  * ca65: The long branches do now accept a numeric address as an argument.

  * cc65: It is possible to define assembler labels using the inline assembler.

  * cc65: A case selector of type unsigned short will no longer cause an
          internal compiler error.

Please note that the libraries and headers do not have changed, so you don't
need to download the machine specific packages, if you have the -beta1 ones.

Sorry for the inconvenience. Please keep the feedback coming!
