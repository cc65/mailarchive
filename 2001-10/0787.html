Hello Ullrich,

UvB> The long branches are an addon done with macros. The main problem in
UvB> implementing them natively is that the generated code is relocatable. This
UvB> means that not all expressions evaluate to constants as with simpler
UvB> assemblers. This means that you cannot simply go back and change anything,
UvB> because that in turn may change expressions all over the code. That does not
UvB> mean that it cannot be done, it does just mean that it is a lot more work
UvB> because of the relocation.
