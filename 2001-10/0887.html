It was me. I have ported LUnix (http://lng.sourceforge.net) to Atari 8-bit in
few days after a flamewar on pl.comp.os.advocacy newsgroup. It is not much
usable as only kernel and micro-shell were working (microshell allows only for
loading a file and reboot) and there is no I/O code (and some more features),
but console handling code (screen and keyboard) is complete.

Since it seems for me too difficult to do a real Atari disk or rs232 stuff I
was thinking about something like 64net/2
(http://sourceforge.net/projects/c64net/) with 4-bit cable instead of 8 or
something. It would be easy because 64net/2 support is already present in
LUnix. Anyway I don't have time for my 8-bit hobbies now so any help will be
welcomed. But I must warn everybody that LUnix does not use any code that is
not from LUnix (including ROMs and any other wedges, add-ons etc.).

Funny thing. Uz started this thread with C= vs. Atari accent right now, when
second C= vs. Atari flamewar rolls through pl.comp.os.advocacy :)
