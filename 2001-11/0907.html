> Ok, but will there be a graphics header sometime, or do I need to make
> it myself? :)

The API discussion and the existing header file were the first steps that
should finally lead to such a library. Since cc65 is a freeware project and no
one gets payed for all the work, there is no time schedule. So, if you have an
urgent need for such a library, your best bets are writing it yourself or help
other people writing it. 

If you are writing code and want it included into the cc65 distribution, your
code has to meet a few (mostly formal) requirements. I do also offer CVS
access for developers, which means that you have direct access to the current
development sources. For security reasons, a static IP address is required for
CVS access.
