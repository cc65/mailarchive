Hi Adam,

"Adam Dunkels" <adam_at_sics.se> wrote:

> On Tuesday 27 November 2001 07:07, you wrote:
> > That's some kind of NAT daemon, right? Well, if it does do NAT, it
> > absolutely *must* to simple IP forwarding/routing. The Windows TCP/IP code
> > is I believe BSD based,

No, it is not a NAT daemon, it is a proxy-based approach.

Windows is not able to do NAT without third party software (for example, 
NAT32, WinRoute, ...)


> ... and since IP forwarding has been in BSD since the 80's, Windows should 
> support it as well.

IP forwarding is supported by the NT line (don't know about the 95/98/Me 
line), but at least on NT, you need to install some drivers which are only
shipped with NT4 server (*not* NT4 workstation).
