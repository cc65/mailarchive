On Tue, Feb 12, 2002 at 07:40:30PM +0100, MagerValp wrote:
> But what's so horrible about GPL:ing an emulator?

This is an old discussion. The GPL has a viral nature, which means that anyone
who wants to use your code must place his own code under the GPL. For one, I
don't like to be forced to use a specific license for my projects, and I don't
want to force other people to use some specific license for their projects.
IMHO people should be free to choose whatever license they do like for their
own code. Second, GPLed code is not usable in some areas, for example if one
wants to use it in a commercial (maybe binary only) product. I'm happy about
anyone using my code, even if they make money with it, so I don't want to use
a license which prohibits such doing.

This is of course just my humble opinion. Other people think that the GPL is
the greatest thing since sliced bread, and that is ok with me, as long as I'm
allowed to have my own opinion. And as I see it for my own projects, the
disadvantes of GPLed code are far greater than its advantages, so I will not
use GPLed code (I have contributed code to other peoples GPLed projects, but
that's a different story).
