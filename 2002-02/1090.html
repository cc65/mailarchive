>>>>> "UvB" == Ullrich von Bassewitz <uz_at_musoftware.de> writes:

MV> But what's so horrible about GPL:ing an emulator?

UvB> This is an old discussion. The GPL has a viral nature, which
UvB> means that anyone who wants to use your code must place his own
UvB> code under the GPL.

Yes, of course, and I'm not with the rabid crew that argues that the
GPL is the One True License or anything. However, modifying someone
elses 6502 emulator saves you an afternoon of work, and it sounded
like you just wanted something basic for testing and debugging.
Writing a 6502 emulator is actually fairly simple (as long as you
don't need decimal mode, undocumented opcodes, or cycle exact beha-
viour) but why reinvent the wheel?
  Well, I'm not arguing either way really, but there are good cores
to use out there. Oh, I forgot about another one erlier, the one by
Marat Fayzullin, which is free for nonprofit use. Check it out at
http://fms.komkon.org/EMUL8/
