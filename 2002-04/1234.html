On Tuesday 02 April 2002 18.55, you wrote:
> I think the GPL is very easy to read and understand. I think it is very
> clear. I also think that it very clearly states that anything which is
> statically linked with their libraries must also be GPL.

No, it says that anything linked (statically or dynamically, it doesn't 
matter) with GPL code must be freely avaliable in source code form. It 
doesn't require it to be under the GPL, but it has to be licensed under a 
license that is compatible with the GPL. There are a number of licenses that 
are GPL-compatible, including the zlib license. Since the zlib license and 
the cc65 runtime license are the same, the cc65 runtime license is 
GPL-compatible.
