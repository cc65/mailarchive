Ok, so the emulator argument does not hold. It's still a lot of work to add
65816 code generation plus runtime and C library support for all the target
systems. Without the help from Christian, Maciej and all the others, it
wouldn't even be possible to support all the 6502 targets we do now have. And
there are still some targets that aren't supported or that have important
features missing. The Apple ][ support is a good example for the latter, and
up to some degree the commodore systems (missing file I/O). BTW: The apple ][
target would need a maintainer. Is anyone interested?
