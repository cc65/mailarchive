> I think there's some redundancy (STACK?) in the
> config file, along with indecision over whether to
> use BSS or DATA for the sprite ram. I assumed
> data as there's no BSS init code (only indirectly).

Since the program is an assembler program anyway, there is no real difference
between BSS and DATA, so you may also remove BSS completely and place any data
into DATA. You may also remove the STACK segment, since it is not used.
Additionally, some segments and memory areas use the "define = yes" statement,
which means that the linker defines symbols for start, end and size. As far as
I can tell, none of these symbols are actually used, so the "define = yes"
statements can be removed.
