Hi,

I like to introduce myself with a wish:

Because the 'Atari-Lynx-adaption' of an former version of cc65
is rather outdated (no ANSI etc.), I like to see the reunion
of the Lynx back to cc65 'mainstream' development.

How can I influence the development in this direction?

Providing the linker config/startup code should not be
the problem, but I would appreciate the support of the
65SC02 processor.
This should not be a big problem, since the SC02 is just a 65C02
(and should also benefit from the enhanced instructions) but without
the single-bit instructions BBR/BBS/RMB/SMB. It's important
that also the assembler dissallow the usage of this instructions
if the 65SC02 target is selected...

Is there any chance for this improvement? Does the compiler
acually uses (in C02-mode) the BBR/BBS/RMB/SMB instructions?

Regards
chrisker
