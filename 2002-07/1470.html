Just as an additional explanation:

The string is passed "as a whole" from the scanner to the parser. Since
translation is sometimes needed and sometimes not (for example, the asm()
statements need an untranslated string), it would be needed to keep track of
the original form of each of the characters in a string. While this would be
doable, it needs a lot of code and has almost no advantages, since the same
thing can be achieved by other means.
