On Thu, Jul 18, 2002 at 04:05:03PM +0200, groepaz wrote:
> look at this, problem is a case value of "0x8000" (min_int) used in a
> switch statement where the switch is done by a signed int.

Ok, I've checked the standard and it says that integer promotions on the
controlling variable should be applied. This would mean that the compiler does
indeed behave incorrectly - it should convert the variable i instead of
complaining. 

I have to admit that I'm not happy about this. The code is unclean, and doing
a type conversion on i for each label that does not have the same type
introduces quite some overhead under the hood. But then, the standard is the
standard is the standard...:-)

Thanks for the report, I will see what I can do about it.
