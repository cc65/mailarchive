On Tue, Nov 12, 2002 at 06:35:26PM -0600, Brian Bagnall wrote:
> Just out of curiosity, is the reason for no standard C file I/O because of
> something to do with the 1541 drive?

The reason for no standard file I/O for the CBMs is that I refused to do it,
and no one else did it either. CBM specific file I/O is there because someone
(Marc Rintsch) did it. And, the Atari has standard file I/O, because Christian
Groessler wrote the necessary low level routines.

In general, a feature is available, if someone is doing the necessary work.
Most problems have a social reason, not a technical one:-)
