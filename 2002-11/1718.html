Now that I know the facts about disk access speed, I'll be using the cbm
disk access methods rather than stdio. The code base can still be portable
using #if defined so it still allows us to achieve our portability goals. It
would be preferable to have stdio just for completeness of ANSI C, but
obviously not a dire need.

On a related topic, the cprintf special character \n should have standard
ANSI functionality. I like the idea of having newline and carriage return,
but I found this confusing as a newbie. When newbies like me are following
'hello world' examples out of books (like the excellent "C Programming
Language" by Kernighan and Ritchie) it can produce unexpected and messy
results with the authors sample code.

I think instead of /n and /r as they are now, it should be:

/n = carriage return and new line
/r = carriage return only
/l = new line only

(or if there is some other more accepted character newline character in
existance)

- Brian
