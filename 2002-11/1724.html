It's not as bad as I thought in the first place: For one, I can give a range
of logical file numbers that are not used by the C library. A program using
CBM file I/O will have to use logical file numbers from this range. This is a
minor limitation I think. Second, after some thought, I doubt that my
"optimization" will really work without problems. For now, I will leave it as
is, and try it only if there are problems with file i/o speed.
