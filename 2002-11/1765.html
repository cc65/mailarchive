Since these routines may be useful for some people (one example would be to
compile code written for the old cc65), how about this:

I can add a "contrib" directory to the FTP server for libraries and stuff that
has no place in the main cc65 distribution, either because of the copyright,
or because it does not fit into the general concept. Just because some code
does not fit into the main distribution does not mean that it is not useful.

My suggestion would be to impose the following rules for stuff that goes into
the contrib directory:

  * It should be in the form of one archive that contains a version number in
    the name, plus one text file that has the same base name as the archive
    with a .txt extension.

  * The text file should state who wrote the code, describe what the
    library/code in the archive does, the license for the code, on which
    platforms it runs, the compiler version it has been tested with, and - if
    necessary - other prerequisites needed.

Note: Sample programs and similar stuff can also go into this directory.
