On Tue, Dec 10, 2002 at 10:00:34AM +0100, Maciej Witkowiak wrote:
> OK. So where this kind of information should go? Certainely not to em.h and
> not to library sources.

I will add a new doc file named "targets.sgml" which will contain target
specific stuff. Some of the information from internal.doc could also go into
this file (available memory for the different platforms and so on). Maybe we
can have a similar structure for the information for each of the target
platforms, so it is easy to find.

It is somewhat unfortunate that the information starts to be cluttered over
several places (some info about em.h should also go into library.sgml, a
function description is already in funcref.sgml and information about the
available drivers will be in targets.sgml), but I see no way to avoid this.

Please allow me to wait until the new version is released before I will add
the new file. This will mean that there is not much information about the
extended memory API and the available drivers (same is true for TGI), but it's
the first release of the new APIs so I think this is acceptable.

BTW: If anyone is willing to spend a little work on the function reference, I
would be quite happy. It is a list of all available functions (standard C
library and cc65 specific things). Because each function is a separate entry,
it is easy to contribute entries for just add two or three functions.
