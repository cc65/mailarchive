It sounds to me that the Linux port of Atari800 is very bad.  The Windows port Atari800Win Plus is very good, but still pretty slow.  You need to run the emulator in a  low resolution on slower machines (like mine.)

As for getting a directory of the disk from BASIC: When you don't boot from a DOS disk, the FMS doesn't get loaded and you can't easily get DOS functions such as disk directory and loading files.  However, you probably don't want to be in BASIC when you run your cc65 programs anyway.  Boot from a DOS disk and load your program from there, or (at least with Atari800Win) you can just run an executable directly via a menu command.
