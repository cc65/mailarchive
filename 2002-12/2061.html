Hello.

I gave the win32 version of cc65 v2.9.0 a spin. I found out that GRC
caused a Windows error and refused to create a VLIR file. I resorted to an
older and working GRC executable for the time being.

Also, the LD65 linker, with a target for GEOS, reported 'segment
"STARTUP" and "LOWCODE" does not exist'. What are these two items?
Apparently, these two errors did not affect the assembly of my dotView
project. Keep in mind that this project is 100% assembly and I'll mind the
memory management details myself.

Also, I forgot to mention a small project I used cc65 for: A patch for
geoCanvas 64/128 to run in Wheels 64/128 systems. The source code file is
at:

http://www.freenet.edmonton.ab.ca/cue/wcanpat.src

Enjoy!
