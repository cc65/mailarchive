On Mon, Dec 23, 2002 at 04:13:46PM +0100, Maciej Witkowiak wrote:
> BTW: there should be BIG FAT warning that modules have to be type USR for C=
> targets.

I know, I know. Thats one of the things I wanted to document before the
release but forgot it. In fact, it's not only drivers but any other file. You
will get an error if the type is not the same as in the _filetype variable
declared in cbm.h. 

I will remove the code that adds the type when the file is opened for reading
for the next release. Currently, the file name parser adds ",t,a" to any file,
and this is the reason for the problem.
