UvB> Sounds like an interesting bug. How can the autostart code mess
UvB> up the floppy directory? I never have interesting bugs like this

It's not cc65-specific, it was just that my testing methodology was
different. With basic I typed it in and ran it in the emu, with cc65 I
drop d64's on VICE.

I haven't received an answer from Andreas yet, so I can only
speculate. But I think the problem is that VICE temporarily turns off
true drive emulation when you use autostart (to load the program
faster). When it turns it back on the 1541 is in a slightly confused
state. If you load something from the drive it seems to get back on
track again.
