> By quoting <http://www.cc65.org/>:
> »C functions may not return structs. Struct assignment is possible.«
>
> I got the impression that return value could not be a struct, but a
> struct a parameter was ok. I propably misunderstud the homepage.

Quoting the docs, which come with each binary package and which are available
online (cited from http://www.cc65.org/doc/cc65-4.html):

  * C Functions may not return structs (or unions), and structs may not be
    passed as parameters by value. However, struct assignment *is* possible.

The cc65 docs aren't perfect (I know they are not), but in this case it
definitely not my omission:-)
