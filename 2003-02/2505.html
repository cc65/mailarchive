And thusly M Ancher spake:
> 
> On Tue, Feb 25, 2003 at 02:05:42PM -0500, Bryan Pope wrote:
> > I think if the line was:
> > 
> > xmemmove (*(wclass->superClasses, *(superClass->superClasses), size);
> > 
> > would fix the problem.
> 
> > P.S.  I am assuming that "xmemmove" has the same prototype as memmove 
> > (void *, void *, size_t)
> 
> No, unfortunatly it doesn't. :(
> 
> The prototype for "xmemmove" is:
> static void xmemmove(uchar *dst, uchar *src, uint32 size)
> 

O!  Then you should cast the destination and source.  If you want to move the
superClasses *list* use:

xmemmove ((uchar *)wclass->superClasses, (uchar *)superClass->superClasses, size);

On the Watcom compilier I use at work the original problem only generates 
a warning.  (Pointer type mismatch)
