On Wed, Feb 26, 2003 at 01:59:14AM +0100, Groepaz wrote:
> it would actually be a smart move of the
> compiler to tell the line of the macro aswell :=P *hint* :=DD

As with most other C compilers this is quite difficult, because macro
expansion is done first, and only the result is passed as text to the scanner.
This means that the compiler itself has no information that a given piece of
code stems from a macro expansion. Of course this could be solved by embedding
special _Pragma()s into the input stream, but this is a lot of work and
probably not worth all the trouble.
