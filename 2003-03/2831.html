>
> BTW, to reduce the [fast-loader's] memory-image that needs
> to be loaded, I'd load the disk-drive code with the & command
> directly from the disk.  Has anyone tried if it works
> on CMD drives?  (The command "&name" should execute
> the file "&name,u" in the disk drive.)

Yes, but the ampersand isn't a part of the file's name, on CMD drives (and
the 1581, too, I think).  You can put a drive-/partition-number in front of
the file-name.  And, you can put a directory-path in front of the name (on
CMD drives).  The BASIC statement

open 15,9,15,"&57/dir1/dir2/:name"

would load-and-run the USR file called "name", which is in the subdirectory
called "dir2", which is in "dir1", which is on the partition that's
numberred fifty-seven.
