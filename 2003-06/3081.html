On Wed, Jun 04, 2003 at 11:33:11PM -0400, Greg King wrote:
> CBM programs are "sticky" (they stay in memory; we can rerun them without
> reloading them).  That means that we should avoid static initializers; and,
> programs should restore their environment, as much as possible.

While I would say that you're right in most cases, I still don't think it is
true for DEVNUM. Since every file operation changes DEVNUM, programs and
utilities for the CBMs do not expect DEVNUM to remain unchanged when a program
is run. And, most C programs cannot be run twice, because static data has
changed.
