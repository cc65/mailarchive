i'm playing a bit with scpu stuff atm and noticed some things which i think 
cant do with ca65, but which are used frequently in sources i check so it 
might be nice to be able to do this in ca65 aswell :=)

1) getting the MSB of a 24bit value.... something like the > operator but 
returning the upper 8 bits of a 24bit range instead. 

2) forcing 16bit absolute opcodes with 8bit arguments.... ie if i wrote "lda 
2" and wanted it to assemble the 3-byte code. (this one would be nice not 
only for 65816 code :=))

3) like 2), but for 24bit addresses. this one is kindof really needed for 
native 65816 programming, else its kindof tricky to work with the first 64k 
segment :)

the scpu assemblers i have checked solve 2) and 3) by certain 
prefixes/operators to the actual expressions, might be worth considering 
something simelar.
