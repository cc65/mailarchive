> Any suggestions?  Anything I'm missing?

You are missing:

3) Link the whole project in one run and tell the linker to produce more than
one output file. This way, the linker will resolve symbols automatically, and
there's no need for any kludges. This is what I've done with Elite128. Here is
a snippet from the linker config that shows that it generates 8 output files:

  MEMORY {
      BASICBUF:	start = $00200, size = $0059, type = rw, file = "";
      CASSBUF1:  	start = $0033C, size = $00C0, type = rw, file = "";
      CASSBUF2:	start = $0033C, size = $00C0, type = rw, file = "_boot";
      LOCODE:   	start = $00700, size = $3900, type = rw, file = "_locode";
      SCRFILL:	start = $07680, size = $0180, type = rw, file = "_hicode", fill = yes;
      HICODE:    	start = $07800, size = $5800, type = rw, file = "_hicode";
      HIDATA:   	start = $0D000, size = $2A97, type = rw, file = "_hidata";
      PLATFORM:	start = $0F880, size = $0200, type = rw, file = "_c64", define = yes;
      REU:	start = $0F880, size = $0200, type = rw, file = "_reu";
      HEAP:	start = $0FA80, size = $0540, type = rw, file = "";
      HI816:   	start = $10000, size = $10000,type = rw, file = "_hi816";
      REUDATA:	start = $00000, size = $10000,type = rw, file = "_reudata";
  }
