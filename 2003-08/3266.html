On Mon, Aug 25, 2003 at 12:15:14PM +0200, Andre Majorel wrote:
> I thought that C99 variable length arrays meant that you could
> define an auto array whose size is not known at compile time (void
> func (size_t n) { char buf[n]; }). Are you sure that they also
> allow arrays whose size is not known at all ? What would you
> expect the total size of struct MyStruct to be ?

There are two different new array features in C99:

    1. Flexible array members, and

    2. variable length arrays.

Christian was talking about the former feature, which is supported by the
current development version of cc65, while you mean the latter, which is
supported - and won't be for some time, since the internal structure of the
compiled code (no basepointer) would make it difficult.
