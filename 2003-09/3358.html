On Fri, Sep 05, 2003 at 10:34:36AM +0200, Groepaz wrote:
> is there another way then that would let me do that with a "custom" memory
> range in zp then? if possible something that would allow me to use my
> variables as before (and not having to convert them to pointers or sth)?

You can define your own variables in an assembler file, reference them as
externals and tell the compiler with "#pragma zpsym" that the symbol is
actually in the zeropage. However, the optimizer will not use this
information, so it will not use the variable with the special zero page
addressing modes.
