Todd Elliott dnia 18 wrz 2003 o 12:55 +0930 napisa³:
> Hello.
> 
> I wonder if the grc/ca65 utility can automatically date/time stamp GEOS
> files? I'd hate to manually modify the date/time each time I compile my
> GEOS project.

Read grc docs - if you want to use a fixed time&date just put 'date' field
to header section of resource file. To make grc always use compilation
time&date comment-out or remove 'date' field.

> Secondly, is there a commandline utility or switch to ca65/grc that will
> insert the newly-created programs into a .d64/.d81 disk image? As it
> stands, when I'm done compiling my GEOS program using ca65/grc, I have to
> use StarCommander to insert these .CVT's into a .d81 disk image. After a
> few debug/compile/edit cycles, this process has become somewhat tedious.

I remember that Joe Forster announced that SC 0.83 has batch processing
capability, you might want to try it.
As for c1541 from VICE hold on until next VICE release as all currently
available versions of this program have a bug in geoswrite command that cuts
off the very last byte.
