Things in GEOS that Contiki could benefit from:
- disk drive turbos
- low level file I/O routines (find file, list directory etc.)
- graphics engine for drawing windows/menus
- pointer provided by OS.

There is 22KB available if GEOS menu/gfx engine would be used.

If using text screen only, we have 34KB (in two chunks: 26+8) available.
Plain CBM conio would replace GEOS graphics then, pointer could still be
managed by OS.

With VDC gfx there is 22+8KB free, with VDC text - 30+8KB.

Is it enough or not?
