On Sat, Oct 11, 2003 at 04:11:40AM +0930, Todd Elliott wrote:
> Still, it would be nice if DA65 could somehow support GEOS disassembly.
> Sure, I could use separate utility to split up the GEOS .CVT file, but it
> does mean an additional step in the process. Then I would have to
> construct a .CFG file for CA65/GRC to compile it again.

Provided there's such a split utility, it is always an option to create a
"split and disassemble" utility that calls the split utility and da65 under
the hood, in a similar way as cl65 does for the other tools. Or combine it
with da65. But the first step would be to have such a smart split utility,
just to see where the problems are.

> A quick glance into the help section of the DA65 disassembler shows that
> it does not support symbol tables.

Are you using one of the 2.9 versions? If so, I would strongly suggest to try
the snapshot version of da65. Versions 2.9 and before were completely
unfinished and lots of features have been added. Most important: The
documentation for 2.9 incomplete and does not describe the info file syntax
(info files were available in the old versions but not documented).
