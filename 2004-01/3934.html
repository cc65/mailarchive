On Mon, Jan 05, 2004 at 10:25:56AM +0100, carlos wrote:
> so an assembly procedure, from which the unsigned char oder unsigned short
> int return value will be printed
> with printf should always clear x-reg before exiting, or?

*Any* character is always promoted to int by the compiler. This means that all
functions returning a char must actually return an int (promote the char to an
int according to its signedness). In most cases (unsigned char) this is done
by clearing the X register before returning. Please note that this has nothing
to do with printing something or with the printf function! Not returning chars
as ints will give errors in other places, too.
