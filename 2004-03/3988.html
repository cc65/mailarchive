Stefan Haubenthal has ported the 3dmaze program to cc65 using TGI. The user
moves within a 3D labyrinth, the goal is to get out of it. The size of the
labyrinth is passed as parameter, or will default to 10x10.

While the game is not a killer as it is now, it could be used as a core for
other, more elaborated games. I've created a first version, so you can have a
look at it:

        ftp://ftp.musoftware.de/pub/uz/cc65/contrib/3dmaze-0.1.zip

The current code runs ok on the C64 (binary is inside the archive), but has
problems on other platforms (like the C128). These problems seem to be aspect
ratio related and I don't think they are difficult to fix.
