A more basic (pun intended) reason not to do it that way is that it
"re-invents the wheel!"  The BASIC and Kernal ROMs do exactly what is
described above, before they run the C program.  There is no good reason to
do it twice ("rem," is not _that_ ugly :-).  It would be a waste of time
and file-/memory-space.
