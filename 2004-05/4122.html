One of the FreeBSD ports maintainers has notified me that there are problems
with cc65 on 64 bit platforms. The tools compile, but translating the
libraries fails. Since this worked some time ago, I don't expect big problems,
but I cannot find the reason by just looking at the code, and the FreeBSD
people were not able to give me any more information (not even about the size
of the data types on the platforms where cc65 fails).

Since I've failed to reproduce the code on one of my SGI workstations, I would
like to ask, if someone with an AMD or Sun 64 bit system can give me temporary
access via internet, so I can find and fix the problem. What I need is ssh
access from a fixed IP, some harddisk space and gcc/gdb installed. Having
something graphical (like kdbg) would be a plus, but is not really necessary.
