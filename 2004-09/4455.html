The problem is that both TurboC and GCC generate a program for a 
platform that uses the same character set as the source code.  So 
there's no translation necessary.

The cc65 tools have to translate the ASCII stuff to the target platforms 
that don't support ASCII.

BTW it's not the compiler but the linker that translates strings.  This 
way you can compile C source to an object file and link it into 
executables for different target platforms and always get the correct 
text output.
