Hi!

I've added code generation for the SWEET 16 pseudo CPU to the ca65 assembler.
For more information about SWEET 16, please see here:

        http://www.6502.org/source/interpreters/sweet16.htm

Apple ][ users should be able to use SWEET 16 out of the box, other systems
are missing the interpreter and the register zero page assignments. Both could
be added as part of the runtime library if requested.

There's a (too) short paragraph in the docs about the new mode.

Please note that I don't have any intentions to add SWEET 16 code generation
to the C compiler:-)

New snapshot binaries are currently building. Have fun!
