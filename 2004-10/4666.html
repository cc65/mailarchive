Hi,

The tgi-library has some primitives that I don't believe should be there.

The point, line and bar are good primitives as they can be created by
hardware (stretching and tilting a pixel in the Lynx).

But the circle does not seem to have any kind of hardware support on any
platforms.

I would like to see a filled triangle there instead of the circle. A
filled triangle can be generated by the Lynx hardware from a pixel that is
tilted and stretched by the hardware.

Another curiosity is the text. I don't believe that the text should be in
the tgi-library. For a system like Lynx I would like to create a text
font. But it should be usable by the text tools like console I/O.

How can they use my text-stuff from the tgi-library? If there is a good
way of doing this then text might be OK in tgi. Especially if some systems
need to swap between text-mode and graphics-mode.

But the circle should be removed in my opinion.
