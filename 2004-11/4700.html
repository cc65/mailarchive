Good Morning (or evening),

I've been busy with lots of stuff .... who hasn't....

Anyway, I know people have been interested in the Apple II disk IO libraries and this is an update....

I've finished the seek R/W tests and they have passed.  There is a slight problem with extending the file to exactly 0xFFFF bytes - the library thinks that is an error code from the call.  I'm going to leave that as it is as I have other functions to test and get through.

I'll be setting up a FTP site to download the code for the build until I can catch up to the latest release.  I've been debating such an action but I think it's best because the code is very alpha.  Once the tests have passed I will submit it for inclusion, unless there is a reason not to.

The last time I've tried to "upgrade" to the current build the compiler had a version dependency of a particular C library - which means I have to upgrade myself.

Thanks
