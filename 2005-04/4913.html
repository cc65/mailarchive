After much research I finally managed to get cc65 2.10.1-1 compiled and 
functioning on my Mac OS X system running OS X 10.3.7.

What I did was use Fink.  I found that cc65 2.9.3 was listed on Fink as 
unstable, so I installed 2.9.3 to make sure I understood the process, 
then modified the process for 2.9.3 to install 2.10.1

Now I find I need to do several other things.

1.  Write an include or lib or whatever to allow AIM 65 to be used as a 
target.

Can anyone point me to a set of files that I can use as an example, 
also, what files are necessary? It doesn't seem to me that all the same 
files that are used (for example for the Apple II) would be necessary 
for the AIM 65.

2.  I need to write and include for the 6522 VIA.  Is anything else 
necessary besides writing the include and putting it with the other 
include files?

3.  Probably the most difficult, find a way to make an object file 
compatible with AIM 65 and a way to transfer the object file from my 
Mac to the AIM 65,

Any help or pointers to the info do do any or all of the above would be 
much appreciated.  I can do most of the work (may have to find some 
help for some of it somewhere) but I just would like to have a pointer 
to what to use as examples or starting points to do the above tasks.

Oh, one more question, how stable is the snapshot 2.10.5 from Apr 6, 
2005?  It seems to me that if it is stable, I could avoid all the 
patching for 2.10.1 and the errors that I might introduce in applying 
the patches.

Thanks ever so much for any and all help and info.
