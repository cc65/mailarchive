I have to admit that I have been thinking about the same things...

It was so fun to code with BLL because of the macros that made 
structured assembly
code possible. Like this. A piece of intelligence in my 1k Othello for 
the Lynx:

    lda length
    _WHILENE
      txa
      clc
      adc step
      tax
      lda board,x
      bit #$fe
      _IFNE
        rts
      _ENDIF
      cmp turn
      _IFEQ
        bra .legalmove
      _ELSE
        inc cnt
      _ENDIF
      dec length
    _WEND
    rts
.legalmove:
   ...

For me this kind of code is easier to read as I am more used to code in C.

A set of macros that allows me to do this would be most welcome. And of 
course
as they are just macros you don't have to use them if you don't want to.

This kind of assembly coding is convenient in making games but perhaps 
not in
writing drivers as it hides the real code and makes debugging more dificult.
