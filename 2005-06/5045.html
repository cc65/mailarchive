My questions ---

When I install the 2.10.1 version and then assemble  using ca65 and 
link using ld65 the linker will locate the include files just fine with 
no additional intervention from me.

When I install the 2.10.5 (20053005) version using the same method as I 
used for 2.10.1, I find that ld65 doesn't locate the include files, and 
I must use the -I PATH option to inform the linker where the include 
files are located.

I'm using Mac OS X 10.3.7 and both versions seem to install just fine, 
except for the one difficulty with the include file path in the linker.

What I am curious about, is, is there something different about the way 
2.10.5 finds or where it expects the include files to be that is 
different from 2.10.1?  If there is a difference, maybe I can modify 
what I am doing to take it into account.
