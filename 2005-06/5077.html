On Wed, Jun 15, 2005 at 10:07:25AM +0200, 42Bastian Schick wrote:
> How about, a dayly diff, and a weekly snapshot to keep bandwidth down ?

Many people aren't willing or able to build the binaries for themselves, so
I'm offering a full set of binary packages, which cause the most traffic. Just
offering a diff wouldn't make a big difference in traffic, at least not for
me. Having diffs is also more work, because several of them have to be
applied, and the number depends on the last snapshot downloaded. There would
also be several snapshots on the server (the weekly one, and the daily one),
this would add to the confusion.

If there is actually a need for diffs, I could add them, but I'm not really
sure if they would be used a lot.
