So maybe the best way is to link all run-time
functions to known addresses and stick with them.
(even the ones that are not to be used by the main
code but that may be needed by the modules). Assuming
that cc65 library will change slower than the main
code of the project, portability between versions will
be easier.

Since some WinXP programs does not run under Win95
(and vice versa) anyway, it shouldn't be a very big
issue, many people still use these OSes :)
