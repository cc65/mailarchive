ca65html incorrectly parses

	bne :-

as a label named "bne". ca65 apparently allows whitespace around the
label name, and the ca65html regexp is a little too greedy. As I don't
use that particular feature myself (does anyone?), I hacked the regexp
to only allow spaces before =, and not before :. I also made it output
valid HTML 4.01 Strict + CSS when colorizing. Patch is here:

  http://www.paradroid.net/tmp/ca65html.patch
