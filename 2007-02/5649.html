Hi Rich,

> it seems to be a huge pain to make it work with inline 6502.

Inline assembler is targeted towards spot optimizing C code, not
writing assembler programs.

> I think this would be the best way to do it..

I think the best way to do it would be to use the cc65 assembler for
your 6502 coding. It's a feature-rich macro cross assembler running on
a lot of platforms.

It generates relocatable objects files you can link together with the
objects files generated by the C compiler thus having a consistent
memory model.

I personally use it for all type of 6502 development. Even totally
unrelated to C and/or the C library.
