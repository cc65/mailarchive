On Sunday 25 March 2007 13:17, Andreas Koch wrote:

> i'm working on a little game with modified charset.
> I noticed CC65 is translating uppercase letters to
> inverted uppercase chars (0xC0+) and lowercase
> letters to (uninverted) uppercase chars (0x40+).

I think you are confusing PETSCII with screen code here.  There are no 
chars in PETSCII for inverted characters but screen codes that display 
inverted characters.  The string constants in CC65 programs are meant 
to be used in IO operations and as file names, not for "poking" 
directly into the screen memory area.
