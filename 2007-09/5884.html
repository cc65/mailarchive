Hi all,

for a university project i just ported a sort of bytecode
interpreter to three platforms for comparison - windows pc,
c64 (cc65) and AVR (winavr/gcc). I thought you might find
the results interesting, but of course this is in no way
a full blown real world benchmark.

I somehow expected the 6502 and the AVR to have a similar
performance per Mhz, but i found AVR to be lots faster.

Numbers i got (interpreted instructions per Mhz and second):

PC (1540 Mhz) : 131  // lots of debug code and windows screen updates
C64 (0.97 Mhz): 236
AVR (10 Mhz)  : 1659

I wonder if the factor 7 between 6502 and AVR is caused by
cc65 vs. gcc performance, or by the processors itself.
(The code basically does some masking and shifting, a large
switch/case and some 32 bit integer multiplication/division).
