On Monday 29 October 2007, Benjamin Bahnsen wrote:

> Currently i make use of the .org control command and create lst-files
> which I parse row by row for memory addresses and bytes. This is quite
> easy. I just have to stop if i find a "rr" byte or come over a memory
> location, which has already been read before. I know that this is not
> the intended way to make use of CA65, but it works good so far.
> Nevertheless it would be nice to have a more common way to create
> binaries. Does someone have an idea? I was thinking of modifying CA65
> to add a switch, which generates a binary (if possible).

The common way is to use a linker config file instead of ``.org``.  
Together with a Makefile you can easily compile and link the project 
with one command.
