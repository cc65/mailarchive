On Thu, Nov 01, 2007 at 01:24:59PM +0100, Benjamin Bahnsen wrote:
> Can anyone explain the linker switch -m? The export list in the generated map file is empty, altough my source contains many
> .export
> declarations.

A normal map file will only contain exports that were actually referenced
(that is, imported somewhere). If you want all exported identifiers in the
mapfile, use -vm (verbose map file) in addition to -m.
