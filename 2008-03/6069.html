>  A small question/suggestion: wouldn't it be worth migrating the code
>  repository to SVN and setting up read-only access for anonymous
> users?

The former is not a requirement for the latter.  I know that
Subversion is better than CVS in pretty much every way, and is the
heir apparent to the open source version control crown for that
reason, but you can set up anonymous readonly access to a CVS
repository with significantly less work than migrating the whole thing
to SVN.
