I have a query about the macro (#define) naming convention in
cc65. In other C compilers I have headers define their presence
with, for example, __alarm_h, __kernel_h, __stdio_h, __ctype_h and
platforms with, for example, __unix__, __arm__, __riscos__,
__i386__, __bbc__.
 
cc65 uses _STDIO_H, _CTYPE_H and __APPLE2__, __BBC__ etc.
 
On and off I've been updating Small-C for the BBC, and that has
included rewriting the modules and headers into a standard format.
Should I be using __stdio_h and __bbc__ or _STDIO_H and __BBC__ ?
 
Previous code I've written with platform-dependant code in it has
used __bbc__ form macros.
