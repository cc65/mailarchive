It may be but it is more difficult than you assume. i++ is not just a
statement, but an expression that results in the value of i before it was
incremented. To determine if the optimization is possible, the compiler has to
check if the result of the expression is not used. It is much easier if the
programmer says explicitly "I don't care about the value before the increment"
by using ++i instead.
