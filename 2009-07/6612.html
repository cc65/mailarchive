On Mon, Jul 20, 2009 at 10:39:05PM +0200, Ullrich von Bassewitz wrote:
> The idea is good, but I don't think the compiler should reference _exit but
> the STARTUP segment instead.

Just found out that this won't work. __STARTUP_RUN__ (or whatever) is
generated by the linker once it finds a startup segment and the "define=yes"
attribute is set. But if the crt0.o module is not linked in, there's no
startup segment. Maybe using exit() wasn't so bad ...
