On Sun, Oct 25, 2009 at 01:44:50PM +0200, Oliver Schmidt wrote:
> That's the ideal solution. Clearing the screen needs to be seen as a
> special case of drawing to the screen - and therefore go the the
> current "drawpage", not "viewpage".

But how did you get the idea, this is not the case? Are there any drivers
implementing tgi_clear as always clearing the view page?

I think one of the problems with the TGI api is that the original C64 driver
written by me using Stephen Judds grlib code is special in several aspects
(only one screen page) and has also missing features (OUTTEXT not
implemented). This driver was more or less the base for all others, which
partially led to todays problems. Maybe we can fix that for the next major
release.
