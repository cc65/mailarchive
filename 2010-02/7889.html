Hi!

Bruce Reidenbach has written a document that nicely explains how to customize
the cc65 library for a new target. It uses an FPFA core as an example, but
most of the text is independent of the actual target platform. It explains,
which files are necessary for platform support, what goes in these files, any
why. The document does also contain some insights into the inner workings of
the cc65 libraries, explains calling conventions and an example on how to
write your own assembler subroutines.

I've placed a copy of the document into the contrib directory of the FTP
server for public review:

  ftp://ftp.cc65.org/pub/uz/cc65/contrib/Customizing%20the%20cc65%20Target.pdf
                            
If you have any comments about the document, please let us know.
