On Wed, Feb 17, 2010 at 08:53:51PM +0100, Oliver Schmidt wrote:
> If the guide targets not only bare-metal (or rather bare-virtual-metal
> in your case ;-)) developers I'd really like to see the zeropage
> memory area size reduced from $100 to $1A. This is the ammount of
> zeropage space used by cc65.
 
For most platforms, this is true, but not for all. $1A bytes is the minimum
size in the standard configuration. Some platforms have more than that space
available and use it for an EXTZP segment.

But you're right, $1A is the common case and it's probably a good idea to use
that.
