The use of zeropage is really completely irrelevant to C-programming. If 
you have enough knowledge to optimize bus cycles we can expect you to be 
able to use the .cfg files.

At least in Lynx programming it is essential to understand segments in 
order to put a cart together. But using zeropage is not necessary at 
all. It is just a way to speed up some minor part of the code. Now when 
I look back at my programming for the last years I have not used 
zeropage to anything at all.

So my view is that the average C programmer does not care if it is 
.zeropage or .segment "EXTZP" as he will not use it. And cc65 is a 
C-compiler. Right?
