> , and everyone knows about them. Those which are maintained outside,

2.13 is out for half a year now, it's maintained (the third bugfix version has
been released a few weeks ago) and it's expected to live for some more time.
If anybody had been offering source and object files, these could have been
used without any changes.

Having an extra source would also show which modules are actually used and
which are not. Just as an example: The CBM510 library had an error in it that
prevented any program from running. No one run about this error because no one
used the platform. So I'm putting work into something that isn't actually
used. I fear the same is true for some of the drivers, and it may be true for
some of the additional modules proposed in this thread. I would be easier with
this if it were proven that the modules are actually used, before including
the into the official source tree (where I have all the work with it).
