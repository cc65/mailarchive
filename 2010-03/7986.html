On Thu, Mar 18, 2010 at 08:01:57PM +0100, Ullrich von Bassewitz wrote:
> A better solution would be to have the linker being able to translate
> symbols.  I've had this idea a few years ago and it can be used to do
> all sorts of nifty things. Simple example:
> 
>         _write  := _ide64_write
>         _read   := _ide64_read
> 
> This would cause the linker to resolve the symbols _read and _write by
> _ide64_read and _ide64_write respectively.
> 
> A more complex example would be
> 
>         * := [ $1_65816, $1_65C02, $1 ]
> 
> which would cause the linker to resolve each symbol "foo" by
> "foo_65816" if this exists, by "foo_65C02" if that can be found, and
> by "foo" otherwise.

that does appear to be the best proposed solution.  now there's just the
simple matter of coding.  ;-)
