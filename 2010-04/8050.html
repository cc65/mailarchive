As there are already people using VS to do CC65 development, I was wondering
if anyone is willing to take on the task of creating a CC65 language service
for VS?  Microsoft has provided an overview of the process as well as
examples:

Overview: http://msdn.microsoft.com/en-us/library/bb166570(VS.80).aspx

<http://msdn.microsoft.com/en-us/library/bb166570(VS.80).aspx>Links to the
examples are in the overview.

I realize that doing a language for C is redundant, but what wouldn't be
redundant is creating a language for CA65 assembler.  This could be
EXTREMELY useful.  Combining a language service and a project template could
allow for full end-to-end development for CC65 developers.  VS Macros could
easily be set up to handle compiling and capturing output and even for
launching VICE to perform testing.

Another cool project (though off-topic for CC65) would be to create a
dialect of BASIC that would tokenize to CBM Basic (version selectable).
 Another cool add-in would be a disk image editor.
