> 1. Does the above hold true when the optimizer is activated?

Yes.

> 2. Does it make sense to presume that the overall codesize of a
> program is reduced with fastcall as the call to the push functions is
> omitted from the call sites (presuming that functions are called from
> more than one site)?

Yes.

> 3. If fastcall isn't beneficial for C functions then why are the C
> library functions written in C prototyped with fastcall?

It is beneficial, since it reduces the size of the program, even when used
with C programs. In case of the library functions, the reasons were:

  * Small reduce in code size
  * Common interface (have all library functions use the same convention)
  * No change in the header when rewriting the function in asm
