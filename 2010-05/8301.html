Yes! I wasn't diving into a discussion of those issues in "our" recent
Makefile threads as they wouldn't have made any difference...

But here very briefly:

- You need to set CC65_HOME to /dir1/dir2/ _without_ a <driveletter>:
prefix. Obviously this means you need to run make from the same drive.

- For Cygwin you need to additionally mount <driveletter>:\dir1 as
/dir1. Then both the compiler and make agree on the path /dir1/dir2/
