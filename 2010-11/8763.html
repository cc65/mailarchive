On Wednesday 03 November 2010, Oliver Schmidt wrote:
> > If we have a small c64 project with just one file and without
> > segments (thus anything goes into CODE), all one needs is
> >
> >        .addr   *+2
>
> I was under the impression that the "BASIC stub" is obligatory - I'm
> just no C64 guy, you see ;-)

It is obligatory if you want to be able to start the resulting binary 
with ``run``.  But even with one file projects you often end 
up "linking" it with some music, graphics, or charsets and compress the 
executable.  And in that scenario the compression programs usually ask 
for the address to start the program and spit out a compressed binary 
with a BASIC line to start the decompression.
