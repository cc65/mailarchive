On Wed, Nov 17, 2010 at 10:46:05AM +0100, Oliver Schmidt wrote:
> I just checked out the c64-asm.cfg and noticed that it uses the
> STARTADDRESS mechanism - as I presumed from following the discussion
> on the list. Now I'm wondering if it wouldn't make sense to have the
> STARTADDRESS adjustable for the default C64 config too - and maybe
> other CBM configs...

I don't think so. BASIC programs (and with the SYS header, the program is one)
must be loaded to the BASIC start, otherwise using RUN won't work. See also
the last sentence in the document that describes c64-asm.cfg:

        http://www.cc65.org/snapshot-doc/c64-4.html#ss4.2

So the case is the same for c64-asm.cfg if the BASIC SYS header is enabled.
