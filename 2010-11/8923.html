> > config too - and maybe other CBM configs...

> I don't think so. BASIC programs (and with the SYS header, the
> program is one) must be loaded to the BASIC start, otherwise using

> 
>         http://www.cc65.org/snapshot-doc/c64-4.html#ss4.2

at $0801 and if it's not skipping the BASIC stub?  I guess not because 
the crts0.o is already compiled and in put in the library file, so it is 
not possible to use conditionally assembly based on the value of the 
start address, right?

The possibility to compile C programs without a BASIC stub to any given 
start adress without the need to write a config file or a new crts0.s 
would be nice.
