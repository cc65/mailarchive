Well a byte buffer of length one is a byte, right!?  So I don't see the 
problem with `cbm_read()`::

  ...

  uint8_t some_byte_value

  ...

  cbm_read(lfn, &some_byte_value, 1);

If you insist on the kernal routines, take a look at the ``cbm.h`` 
header file.  The kernal function names start with `cbm_k_*`.
