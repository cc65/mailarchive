Can somebody explain me the logic behind the following behaviour:

.repeat 10, xyz
    .assert .const(xyz), error
    .byte .sprintf("foo%d", xyz)
.endrepeat

The .assert doesn't trip, but I get this error in the .byte line:

foo.inc(3): Error: Constant expression expected

If I replace "xyz" with a constant number (in .sprintf) the code
compiles. Is this a bug or a feature, and does a workaround exist? :)
