On Sun, Jul 17, 2011 at 03:03:51AM +0000, Tilmann Hentze wrote:
> There has been made a change to linker configuration file syntax in the sense
> that a type for symbol can be now "export/import/weak", where as it was before
> only possible to attribute it as "weak = yes/no".

It's not the only change. There were also some incompatible changes to the
command line options of the tools.

> This makes some problems in existing source code. How could one detect the
> ld65 version and change the linker configuration file accordingly?

Above change is currently in the development snapshot. While it is possible to
track all these changes, I would say it is rather difficult to adjust the
linker config or command line automatically. If you want to try anyway:
All tools output the version number when using -V.

I have to admit that the situation is not perfect, but things have to change
in order to improve. A new major release would probably help, but there are
always some unfinished pieces causing me to delay a new release.
