A - maybe not too related - question: does it make any sense to support
other CPUs, like DTV's "CPU" (not counting the undocumented - but sometimes

I just ask this, because implementing that is not so hard, maybe I can do it
(yes, maybe you can remember that I promised some Z80 support: it seems it
can be harder - for me at least - than I inmagined), but I am curious that
this patch is worth to try to be made at all, ie will it be accepted by you.

I am not sure if "none" CPU is supported now (we talked about that too): not
having any opcodes etc, can be useful when you "compose" only data, so there
are no reserved keywords (I can use macros like LDA, etc) in that case. For
sure, that's the most easy to implement :)
