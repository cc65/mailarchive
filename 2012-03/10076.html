> You talked about the memory limits so a question arises for me, which
> memory "driver" that comes with cc65 is compatible with GEOS?

I don't know. However GEOS usually uses REU / geoRAM on its own as
disk cache. In case this can't be turned off the corresponding cc65
drivers would of course be _not_ compatible with GEOS. Apart from that
I'm unsure if usage of the RAM in question via a cc65 driver is really
to be prefered over the implicit usage via GEOS.
