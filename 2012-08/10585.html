Ok, looking at the apple2auxmem.s file, and it looks pretty
straight-forward, but here are some problems I see right off the bat, all
caused by the issue of the HUGE memory banking window for the Atari memory
expansion ($4000-7FFF).

1. The driver needs to avoid this memory window, so what memory segment do
you load it into?  It would basically need to be in LOWCODE.  BSS probably
will not work, as that's going to be smack in the middle of the banking
window usually.

2. You have to define a 256-byte "window" to use with the EM driver.  This
also is in BSS... same problem as above, how do we avoid the banking window?

Now I'm starting to remember why I gave up... :)
