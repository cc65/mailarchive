Okay, let's try to sort this out. I originally started to call the
beast unit because the variable I want to get rid of is called
_curunit. Now I seem to understand that I'm talking about the device
number.

So rephrasing it:

My current proposal is to re-interpret the filename

"1:name"

as not refering to drive 1 on/in the current device but rather as
refering to device 1. To my understanding this cannot cause any issue
with filenames because "1:" would have been treated by the device
specially anyway.

The obvious restricting is that it isn't possible anymore to refer to
drive 1 this way anymore. To do this one would have to use

"8:1:name"

which would refer to drive 1 on/in device 8.
