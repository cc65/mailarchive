> 
> Just to be sure - this is an issue specific to the directory read
> routine. It is in general possible to successfully open several files,
> right?

Correct. The directory read routine is special, as there is no file for
it. It is only this special handling that fails.

The 154x and 157x drives can handle up to 3 simultaneous open files
(except the directory), with one exception: A relative file counts for
two files, making it one relative file and one sequential ("normal")
file only.
