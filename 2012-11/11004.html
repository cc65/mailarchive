I'm not at home currently, so I cannot check, but at least I can say that the
multiplication is not the problem. If you try to allocate more than 65535
bytes of memory, you have a problem in your program. Having calloc check that
won't help you.

> Background of the code snippet: Of course both values are not hard coded

> struct.  I let the user input a count and want to tell him the count is
> too high if `calloc()` fails.  As there is no function in the C standard

course.
