> This is probably a silly question but do you know of a way that I can
> somehow tell the linker to address "Start:" in crt0.s to $D3B6 thus
> eliminating the segmentation I'm doing with "STARTUP":

No, sorry. There are other solutions, but I know of none which is better. BTW,
that's not really a cc65 problem. If you have just 12K and need some code to
go somewhere in between, the result is always a fragmentation of the memory
space.

Since the startup code won't fill the upper 3 kilobytes, you could place
another memory area above and move some of the stuff there, that is currently
in ROM. That might free up enough space in ROM to be able to place the debug
routines there. They're 6K plus some conio stuff that is needed, so they won't
go above STARTUP. But having them is fun, because they implement a
"fullscreen" debugger for the 6502.
